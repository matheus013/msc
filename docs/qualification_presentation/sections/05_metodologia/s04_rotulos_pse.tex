% Metodologia -- Módulo 5 -- Rótulo e PSE
\begin{frame}{Do benchmark à recomendação}
  \AipePipelineContext{5}
  \keymessage{O rótulo nasce da simulação; o PSE aprende o padrão.}
  \vspace{0.25em}

  \begin{columns}[T,onlytextwidth]
    \column{0.49\textwidth}
      \begin{enumerate}
        \footnotesize
        \item Simulação avalia cada política candidata.
        \item O rótulo é a política viável de menor CTI.
        \item O PSE aprende a mapear características da série para esse rótulo.
        \item Para uma nova série, o PSE recomenda uma política e um ranking.
      \end{enumerate}
    \column{0.51\textwidth}
      \footnotesize
      \[
        y_i =
        \arg\min_{\pi \in \mathcal{P}:\overline{\mathrm{NS}}_{i,\pi}\geq 0{,}70}
        \overline{\mathrm{CTI}}_{i,\pi}
      \]
      \vspace{-0.4em}
      \[
        f_\theta(\mathbf{x}_i) \rightarrow \hat{\pi}_i
      \]
      % \begin{alertblock}{Estágio atual}
      %   \footnotesize Regra de rotulagem e pipeline definidos; validação nacional do PSE é continuidade.
      % \end{alertblock}
  \end{columns}
  \note{
    Este slide unifica benchmark, rótulo e PSE. Primeiro, a simulação avalia
    cada política candidata para a mesma série loja-produto. Depois, o
    rótulo é definido como a política viável de menor CTI sob NS maior ou
    igual a 0,70. O PSE aprende a relação entre características
    $\mathbf{x}_i$ e rótulo $y_i$. Para uma nova série, ele recomenda uma
    política e um ranking; ele não gera o rótulo nem substitui a simulação
    usada para construir a base supervisionada.
  }
\end{frame}
